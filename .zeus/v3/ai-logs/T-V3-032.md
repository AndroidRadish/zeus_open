# T-V3-032 修复测试策略：禁止不加区分跑全量测试

## 问题发现

在 T-V3-031（纯前端 UI 精简）执行过程中，为了符合 AGENTS.md 的"测试全部通过"要求，跑了一遍 `test_v3_core.py` + `test_v3_api.py` 共 45 个测试（耗时 64 秒）。其中包含并发测试、队列持久化测试、端到端调度测试等，与本次纯前端改动没有任何因果关系。

这是流程上的严重浪费。

## 根因

- AGENTS.md 完成定义中写的是"测试全部通过"
- PROMPT.md 写的是"如有 typecheck / lint / test 配置，运行并保证通过"
- 两者都没有区分"相关测试"和"全量测试"

## 修复内容

1. **AGENTS.md**
   - 将完成定义中的"测试全部通过"改为"**相关验证通过**（根据测试范围矩阵只跑必要验证）"
   - 在「验证循环工作法」中新增「**测试范围矩阵**」
   - 明确各类修改对应的验证边界

2. **workspace/base.py 的 PROMPT 模板**
   - 将"如有 typecheck / lint / test 配置，运行并保证通过"升级为"**测试必须遵循范围矩阵，禁止不加区分跑全量测试**"
   - 列出前端/后端/DB/文档四类修改的最小验证集

## 改动文件

- `AGENTS.md`
- `.zeus/v3/scripts/workspace/base.py`
- `.zeus/v3/task.json`

已完成。
