# v3 M-010 — Unified CLI (`zeus_runner.py` v3 Support)

## Decision Rationale

M-010 aims to make `zeus_runner.py` a unified compatibility frontend for both legacy (v1/v2) and v3 projects. The most important missing pieces were `plan` and `run` (now `run_wave`) support for v3. With this change, users can continue using the same familiar CLI commands regardless of which Zeus version their project uses.

## Execution Summary

### Modified files
- `.zeus/scripts/zeus_runner.py` — v3 integration for `plan` and `run_wave`
  - `plan()` now branches to `_plan_v3()` when `--version v3` is passed
    - Reads pending tasks from the v3 SQLite database
    - Groups them by wave and prints the execution plan
  - `run_wave()` now branches to `_run_wave_v3()` when `--version v3` is passed
    - Delegates to `python .zeus/v3/scripts/run.py --project-root . --max-workers 3`
    - After subprocess finishes, prints the v3 status summary
  - `_status_v3()` already existed; `plan` and `run` complete the trio

### Verification
- `python .zeus/scripts/zeus_runner.py --version v3 --status` → v3 DB status ✅
- `python .zeus/scripts/zeus_runner.py --version v3 --plan` → pending task plan ✅
- `python .zeus/scripts/zeus_runner.py --version v3` → delegates to `run.py` ✅
- `50/50` v3 tests passed
- Legacy `main` version commands remain unaffected

## Target Impact

- **developer_adoption_rate** ↑↑↑ — Same CLI workflow across all Zeus versions eliminates context switching
- **backward_compatibility** ↑↑ — Existing v2 users can migrate to v3 without learning a new CLI
