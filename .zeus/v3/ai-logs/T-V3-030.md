# T-V3-030 修复 Dashboard 依赖图渲染（SVG/Mermaid/ECharts）

## 任务目标
修复 WorkflowGraphPanel 中 SVG 空图、Mermaid 和 ECharts 仅显示原始文字的问题。

## 问题分析

用户反馈 Dashboard 的 Graph 面板存在三个问题：
1. **SVG 显示为空图片**：当任务列表为空时，`workflow_graph.py` 生成的 SVG 尺寸极小（280x60）且只有黑色背景，与 Dashboard 深色背景融合后看起来完全是空的
2. **Mermaid 显示为文字**：`WorkflowGraphPanel.vue` 用 `<pre>` 标签直接显示 Mermaid 源码，没有调用 `mermaid.run()` 进行渲染
3. **ECharts 显示为文字**：同样用 `<pre>` 标签直接显示 ECharts JSON 数据，没有使用 `echarts.init()` 渲染图表

## 执行步骤

1. **安装依赖** → 验证：`npm install mermaid echarts` 成功
2. **重写 WorkflowGraphPanel.vue** → 验证：
   - 使用 `mermaid` 库动态渲染 Mermaid 图表（dark 主题）
   - 使用 `echarts` 库渲染力导向图（force layout），支持 roam、tooltip、legend
   - 保留 SVG 的 `<img>` 展示方式
3. **修复后端 SVG 空状态** → 验证：
   - `workflow_graph.py` 的 `to_svg_native()` 在 `tasks` 为空时返回带 "No tasks available" 提示的 SVG
4. **补充 i18n** → 验证：`graph.empty` 键已添加到 zh.json 和 en.json
5. **构建 Dashboard** → 验证：`npm run build` 成功，静态资源输出到 `api/static/`
6. **运行测试** → 验证：45/45 测试通过（v3_core + v3_api）
