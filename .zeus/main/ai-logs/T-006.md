# T-006 AI Log — 全量文档一致性修复与收尾验证

## Decision Rationale

1. **文档必须与实现对齐**：zeus-open 的核心价值是"通用 AI-CLI"，但 README 和 skills 中仍大量引用 `generate-tests.sh`、`zeus-runner.sh` 等 claude 专用 bash 脚本。这会让 Windows/Kimi 用户产生困惑，必须全部修正为 Python 脚本引用。
2. **一次性批量修复优于分散修改**：T-006 作为文档 wave，集中扫描所有 `.md` 文件中的命令示例和脚本引用，确保没有遗漏。
3. **状态文件必须反映真实进度**：task.json、roadmap.json、evolution.md 是 Zeus 框架的"事实来源"，在功能代码完成后必须同步更新，否则 runner 的状态报告会失真。
4. **清理临时产物**：开发过程中产生的 `collected-metrics-*.json` 和 `__pycache__` 不应进入版本控制，通过 `.gitignore` 和 `git rm --cached` 进行清理。
