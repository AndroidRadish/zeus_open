# T-027 — Update Web UI with Phases tab, phase badge, and phase-aware wave selector

## Decision Rationale

A new backend endpoint (`/phases`) is only valuable if users can discover and interact with it. The Web UI needed a dedicated Phases tab, a visible current-phase indicator, and a wave selector that filters by phase to prevent the "1~50 wave dropdown" problem.

## Execution Summary

- Modified `web/index.html`:
  - Added new tab **阶段 / Phases** with a dedicated icon.
  - Added `phases`, `expandedPhases`, and `selectedPhaseId` to Vue state.
  - Implemented `fetchPhases()`, `togglePhase()`, `phaseStatusText()`, `phaseStatusClass()` helpers.
  - Rendered phase cards with status badge, progress bar, localized summary, and expandable nested milestones.
  - Added a **Phase badge** in the header next to the Wave badge (e.g. `P-002 智能与完善`).
  - Upgraded the Dashboard wave selector to a **Phase → Wave**联动 selector:
    - Dropdown 1 selects the phase.
    - Dropdown 2 only shows waves belonging to that phase.
    - Changing phase auto-resets wave to the phase's first wave.
  - Wired `fetchPhases()` into `refreshAll()` and tab watcher.
- Commit: `f7422ae`

## Target Impact

- `ui_usability` ↑↑: no more scrolling through 50-wave dropdowns.
- `developer_adoption_rate` ↑: immediate visual context about which "big push" the project is in.
