## Decision Rationale

T-034 的目标是对 T-030~T-033 进行收尾验证，确保：
1. Global Scheduler、Mailbox、Agent Logs 都有端到端测试覆盖。
2. `main` 轨道不受 v2 新文件/新 schema 的影响。
3. 开发者文档 `skills/zeus-execute-v2.md` 及时更新，避免知识断层。

测试策略：
- API 层：通过真实 uvicorn server 测试新端点。
- Orchestrator 层：通过 patch `MockSubagentDispatcher.run` 保留完整 agent bus 双写流程，同时让 `run_global` 能跨 wave 完成调度。
- 兼容性：直接读取 `.zeus/main/task.json` 验证 legacy 结构未被污染。

## Execution Summary

1. **`test_api_global_status_mailbox_and_agent_logs`**
   - 启动真实 FastAPI server
   - 构造含 `quarantine` 和运行中事件的 task.json
   - 断言 `GET /global/status` 返回正确的 running/pending/quarantine
   - 断言 `GET /mailbox/...` 能读取 mailbox JSONL
   - 断言 `GET /agents/.../logs` 能读取 activity + reasoning

2. **`test_main_track_unaffected`**
   - 在临时目录创建 `.zeus/main/task.json`
   - 验证 `LocalStore` 能正常读取，且没有 v2-only 的 `quarantine` 等字段泄漏

3. **`test_run_global_end_to_end`**
   - patch `MockSubagentDispatcher.run` 在返回前更新 `passes: true`
   - 调用 `orch.run_global()`
   - 验证 3 个跨 wave 任务全部被调度
   - 验证 `zeus-agent-T-001` 的 agent-specific logs 存在（测试了 T-032 的双写）

### 文档更新 (`skills/zeus-execute-v2.md`)
