# T-032 — Agent-level isolated ai-logs directory structure

**Commit SHA:** `1b02a1d6d6e59104d7416e891b103d0f92deb1a6`

---

## Decision Rationale

在引入 GlobalScheduler（T-030）和 Mailbox（T-031）后，日志从“按 wave 聚合”转向“按 agent 隔离”是必然趋势：
- **可追踪性**：每个 Agent 有自己独立的 `activity.md`（人类可读）和 `reasoning.jsonl`（机器可读），便于问题定位。
- **向后兼容**：保留 `wave-{n}-events.jsonl` 和 `wave-{n}-discussion.md` 作为聚合视图，不破坏现有 API 和 Web UI。
- **最小侵入**：`AgentBus` 在 `emit` 和 `post` 时双写；调用方只需传入 `agent_id`。

---

## Execution Summary

### 修改的文件

1. **`.zeus/v2/scripts/agent_bus.py`**
   - `__init__` 接受 `agent_id`；当提供时，初始化 `_agent_dir`、`_activity_file`、`_reasoning_file`。
   - `emit()` 双写：legacy `wave-{wave}-events.jsonl` + agent-specific `reasoning.jsonl`。
   - `post()` 双写：legacy `wave-{wave}-discussion.md` + agent-specific `activity.md`。
   - 新增 `get_agent_events()` 和 `get_activity()` 读取 agent 专属日志。

2. **`.zeus/v2/scripts/zeus_orchestrator.py`**
   - `dispatch_task()` 中构造 `agent_id = f"zeus-agent-{task_id}"`。
   - 创建带 `agent_id` 的 `AgentBus`，替换原 `bus` 传递给 dispatcher 和 bootstrap。
   - `run_global()` 中 quarantine 的 workspace 路径同步使用新的 agent_id 命名。

3. **`.zeus/v2/scripts/test_agent_bus.py`**
   - 新增 `test_agent_specific_events_and_activity`。
   - 新增 `test_both_legacy_and_agent_logs_exist` 验证双写。

4. **`.zeus/v2/scripts/test_orchestrator.py`**
   - 新增 `test_dispatch_task_creates_agent_specific_and_legacy_logs` 验证端到端双写行为。

Agent 级别日志隔离提升了可观测性，为 T-033（Web UI Agent Logs 浏览器）打下基础。通过保持 legacy wave logs，确保了 **developer_adoption_rate** 不因破坏性变更而下降。
