# T-015 — Create subagent_dispatcher.py with ABC + Mock + Kimi + Claude + Auto implementations

## Decision Rationale

To turn `dispatch_task` from a mock into a true unattended runner, we needed an abstraction layer that can delegate to platform-specific AI CLIs without leaking CLI details into the orchestrator.

Key design decisions:
- **Abstract base class (`SubagentDispatcher`)**: keeps the orchestrator scheduler loop clean and testable.
- **`_CliDispatcher` mixin**: handles common concerns (stdout streaming to file, timeout, uniform bus event reporting) so Kimi and Claude implementations only need to supply their command-line construction logic.
- **Auto detection**: `AutoSubagentDispatcher` uses `shutil.which` to pick the best available backend, minimizing configuration friction.
- **Stdout capture**: every CLI run streams combined stdout+stderr to `{workspace}/{task_id}-stdout.txt` so humans can inspect what the subagent did without reading JSONL events.
