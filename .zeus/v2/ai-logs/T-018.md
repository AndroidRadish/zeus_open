# T-018 — Update Web UI Agent Monitor to show last_seen and simple activity indicators

## Decision Rationale

The original Agent Monitor only displayed agents whose `state == "running"`. Because `MockSubagentDispatcher` (and even real CLI dispatchers on fast tasks) completes quickly, the panel was almost always empty, giving users the impression that nothing was happening.

To make the monitor useful, we needed:
1. A **running** section for currently active agents.
2. A **recent activity** section showing the last 10 completed/failed agents with timestamps and durations.
3. Minimal UI changes within the zero-build constraint.

## Execution Summary

- Updated `.zeus/v2/scripts/zeus_server.py` `/agents` endpoint:
  - Now parses `task.started`, `task.completed`, and `task.failed` events.
  - Returns two lists: `running` (active agents) and `recent` (last 10 finished agents sorted by `finished_at` desc).
  - Each agent record includes `agent_id`, `task_id`, `started_at`, `status`, and optional `finished_at`.
- Updated `.zeus/v2/web/index.html` Agents tab:
  - Refactored into two sections: "运行中" and "最近活动".
  - Running agents keep the existing card layout with pulsing cyan badge.
  - Recent agents render in a compact table with status badges (emerald for completed, rose for failed) and a computed duration column.
  - Added `formatDuration(startedAt, finishedAt)` helper.
- Updated `.zeus/v2/scripts/test_integration.py` to assert against the new `running` key instead of the old `agents` key.

## Target Impact

- `ui_usability` ↑↑: users can now see both live activity and a history of what just finished, eliminating the "empty monitor" problem.
- `developer_adoption_rate` ↑: improved trust in the parallel orchestration system because execution is visibly traceable.
