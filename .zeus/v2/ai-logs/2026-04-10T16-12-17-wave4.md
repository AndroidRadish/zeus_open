## Decision Rationale

### Parallel Agent Execution for Wave 4
Consistent with the v2 multi-agent design, T-008 (tests) and T-009 (docs) were delegated to independent `Agent(coder)` instances and executed in parallel. T-010 (validation) was held as a human-in-the-loop gate until both predecessors completed, matching the wave-approval model we ship.

### Integration Test Strategy
Instead of only mocking, we mix:
- **Heavy unit-level integration**: `test_end_to_end_two_wave_execution` uses a real `ZeusOrchestrator` against a temporary project tree, verifying workspace creation, JSONL events, Markdown discussion, and wave sequencing.
- **Real-server smoke test**: `test_api_smoke_full_lifecycle` spins up `uvicorn.Server` in a background thread on a free port, exercising the full REST surface (`/status`, `/wave/{n}`, `/agents`, `/events`, `/discussion`, `/graph/mermaid`, `/wave/{n}/approve`). `/graph/svg` is checked defensively (503 tolerated when Graphviz `dot` is absent).

### Documentation Placement
`skills/zeus-execute-v2.md` follows the existing skill playbook convention. README updates were kept additive: no existing quick-start flows were broken, preserving backward compatibility.
