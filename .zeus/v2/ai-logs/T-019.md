# T-019 — Update README and zeus-execute-v2.md with subagent setup instructions

## Decision Rationale

A new feature (subagent dispatcher) is only valuable if users know how to enable it. The primary entry points for documentation are `README.md` (first impression) and `skills/zeus-execute-v2.md` (execution reference). Both needed clear, concise instructions on how to switch between Kimi, Claude, and mock modes.

## Execution Summary

- Updated `skills/zeus-execute-v2.md`:
  - Added "Subagent Dispatcher Configuration" section after CLI usage.
  - Documented the optional `subagent` block in `config.json`.
  - Included a table of supported `dispatcher` values (`auto`, `kimi`, `claude`, `mock`) with their behaviors.
  - Added safety notes about `kimi --print` implicitly running in YOLO mode and the need for workspace isolation.
- Updated `README.md`:
  - Added "Subagent Dispatcher" bullet to the "What's New in v2" list.
  - Inserted a brief dispatcher config snippet in the "v2 Parallel Mode with Web Dashboard" section, pointing readers to `zeus-execute-v2.md` for details.

## Target Impact

- `developer_adoption_rate` ↑: lowers the barrier for users to switch from mock mode to real unattended execution.
- Reduces support burden by making configuration discoverable from the two most-read documents in the project.
