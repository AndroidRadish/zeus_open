# T-013 — Update Web UI dashboard to display rescheduled task badges and corrected wave progress

## Decision Rationale

When tasks are dynamically rescheduled across waves, the Web UI must surface this to the user so the dashboard remains trustworthy. Without visual indicators, a human might be confused why a task originally planned for Wave 6 appears in the Wave 5 task list.

We chose minimal, non-intrusive UI additions:
- A small cyan badge on the task row showing the original wave.
- A subtle subtitle on the wave progress bar when any rescheduled tasks are present.
- Graph tooltip enrichment to show rescheduling context on hover.

These changes fit within the existing zero-build constraint (pure Vue 3 + Tailwind in a single HTML file).

## Execution Summary

- Updated `.zeus/v2/scripts/zeus_server.py`:
  - `/wave/{n}` endpoint now returns `original_wave`, `scheduled_wave`, and `rescheduled_from` for each task.
  - `/wave/{n}/approve` endpoint now uses `original_wave` (not `wave`) to determine whether all tasks belonging to a wave have passed. This fixes a bug where a task rescheduled into an earlier wave would incorrectly block approval of its true home wave.
- Updated `.zeus/v2/web/index.html`:
  - Added `hasRescheduledTasks` computed property.
  - Wave progress bar shows "（含提前调度）" suffix when rescheduled tasks are detected.
  - Task table renders a cyan "原 Wave X" badge whenever `original_wave !== scheduled_wave`.
  - Graph tooltip appends rescheduling info (`原 Wave X → 现 Wave Y`) to the body text.
- All 12 server tests and full v2 test suite (35 tests) pass.

## Target Impact

- `ui_usability` ↑↑: users can immediately see why a task appears in a different wave.
- `developer_adoption_rate` ↑: trust in the system is preserved because scheduling decisions are transparent.
