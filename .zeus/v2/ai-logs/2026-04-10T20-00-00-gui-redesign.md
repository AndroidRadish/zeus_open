## Decision Rationale

用户对 PyQt 桌面 GUI 提出了两个核心痛点：视觉风格过时（"丑的一坨"），以及与 Web 控制台同时打开时严重卡顿。经过分析，卡顿主要源于三个因素：

1. **主线程阻塞**：原 `zeus_gui.py` 的 `refresh()` 直接调用 `requests.get`，导致 UI 在每次网络请求期间冻结。
2. **控件高频重建**：`AgentsTab.update_data()` 每 2 秒删除并重建所有 Agent 卡片，造成大量不必要的布局计算和内存抖动。
3. **双重轮询竞争**：PyQt GUI 和浏览器 Web UI 各自每 2 秒向后端轮询，当两者同时运行时对 FastAPI 形成双倍负载。

设计决策：
- **保留 PyQt6 而非嵌入 Web 视图**：嵌入 QWebEngineView 会引入第二个 Chromium，反而增加资源消耗。
- **全局暗色 QSS 主题**：采用与 Web UI 一致的 `#0a0a0f` 背景、玻璃拟态卡片（`#111118` + 半透明边框）和青色 `#06b6d4` 强调色。
- **侧边栏导航**：用 `QStackedWidget` + 自定义 `NavButton` 取代原生的 `QTabWidget`，呈现现代 SaaS 仪表盘风格。
- **后台线程化**：引入 `ApiWorker(QRunnable)` + `QThreadPool`，所有 HTTP 请求异步执行，通过 `pyqtSignal` 回写主线程。
- **智能缓存与懒加载**：
  - 轮询间隔从 2000ms 延长至 5000ms
  - 窗口最小化时自动暂停轮询
  - Agent 卡片使用字典缓存，仅做增量更新
  - Graph SVG 按 wave 缓存，仅在切换标签且 wave 变化时重新加载
  - Discussion 仅在切换标签时加载，不再参与后台轮询

## Target Impact

- **developer_adoption_rate**：桌面 GUI 的视觉一致性和响应速度提升，降低了开发者因"界面卡顿/丑陋"而转用纯 CLI 或外部浏览器的心理门槛。
- **ui_usability**：暗色主题与 Web UI 对齐，侧边栏导航更符合现代仪表盘习惯；后台线程彻底消除了 UI 冻结。
- **multi_agent_efficiency**：减少无效轮询和控件重建后，CPU 占用显著下降，与 Web 控制台并行运行时不再"卡成狗"。
