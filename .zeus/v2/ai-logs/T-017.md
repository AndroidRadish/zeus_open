# T-017 — Add unit tests for dispatcher command construction, auto-selection, and timeout

## Decision Rationale

The subagent dispatcher is the new execution backend for Zeus v2. Because it touches subprocess creation, stdout streaming, timeout handling, and platform-specific command construction, it required dedicated unit tests beyond the existing orchestrator integration tests.

## Execution Summary

- Tests were added as part of the `test_subagent_dispatcher.py` file (shipped alongside `subagent_dispatcher.py` in commit `a0c996b`).
- Coverage includes:
  - **Command construction**: `test_kimi_build_command` and `test_claude_build_command` assert correct CLI flags.
  - **Auto-selection**: `test_auto_selects_kimi_when_available`, `test_auto_selects_claude_when_kimi_missing`, and `test_auto_falls_back_to_mock_when_no_cli` verify `AutoSubagentDispatcher` logic under patched `shutil.which`.
  - **Timeout**: `test_cli_dispatcher_timeout` confirms that a long-running subprocess is killed after `timeout_seconds` and returns `exit_code: -1`.
  - **Stdout capture**: `test_cli_dispatcher_captures_stdout` runs a real `python -c "print('hello')"` subprocess and verifies the stdout file contents.
  - **Factory function**: `test_build_dispatcher_*` tests cover all four dispatcher modes and timeout configuration.

## Target Impact

- Ensures the dispatcher layer is safe to run in CI and on developer machines where `kimi` or `claude` may or may not be installed.
- Prevents regressions in command-line construction that could break subagent spawning on future changes.
