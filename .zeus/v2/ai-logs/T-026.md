# T-026 — Extend roadmap.json schema and add GET /phases endpoint

## Decision Rationale

Wave numbers grow indefinitely in long-running projects, making the Dashboard hard to read. Milestones already group tasks by functional module, but humans also need a higher-level "delivery batch" narrative. The Phase layer sits between Version and Milestone, providing named containers (P-001, P-002...) with summaries and wave ranges.

## Execution Summary

- Modified `.zeus/v2/roadmap.json`:
  - Added top-level `phases` array with 3 phases:
    - `P-001 v2 Foundation` (M-001~M-004, waves 1-4)
    - `P-002 v2 Intelligence & Polish` (M-005~M-008, waves 5-8)
    - `P-003 v2 Global Orchestrator & Agent Collaboration` (M-009~M-010, wave 9+)
  - Each phase includes `title`, `title_en`, `title_zh`, `summary`, `summary_en`, `summary_zh`, `milestone_ids`, `wave_start`, `wave_end`, and `status`.
- Modified `zeus_server.py`:
  - Added `_read_roadmap_json()` helper.
  - Added `_compute_phase_status()` based on linked milestone statuses.
  - Implemented `GET /phases` endpoint that joins `roadmap.json` with `task.json`, computes per-phase `status` and `progress_percent`, and returns nested milestones.
  - Enhanced `GET /status` to include `current_phase` derived from `current_wave` and phase wave ranges.
- Commit: `f7422ae`

## Target Impact

- `developer_adoption_rate` ↑↑: humans can reason in phases instead of raw wave numbers.
- `ui_usability` ↑↑: scalable navigation for projects with 20+ waves.
