# T-033 — Web UI Global Execution, Agent Collaboration, and per-agent log browser

**Commit SHA:** `484c2a4e07010a705df3dfec16e726a782db8b5f`

---

## Decision Rationale

T-033 是 T-030~T-032 的前端落地任务，需要把后端新增的 3 大能力暴露为可交互的 Web UI：
- **Global Execution**：跨 wave 任务状态、待执行队列、隔离区可视化。
- **Agent Collaboration**：基于 Mailbox API 的实时消息流，支持标记已读。
- **Agent Logs**：基于 Agent-centric 日志结构的浏览器，下拉选择 agent 后展示 `activity.md` + `reasoning.jsonl`。

UI 风格延续现有 glassmorphism + dark theme，3 个新 tab 与原有 tab 体系保持一致。

---

## Execution Summary

### 后端 API (`zeus_server.py`)

1. **`GET /global/status`**
   - 读取 `task.json` 和当前 wave 的 `wave-{n}-events.jsonl`
   - 返回 `running`、`pending_by_wave`、`quarantine` 三个字段

2. **`GET /agents/{agent_id}/logs`**
   - 读取 `.zeus/v2/agent-logs/{agent_id}/activity.md` 和 `reasoning.jsonl`
   - 返回 Markdown 原文 + 解析后的 JSONL 事件数组

### 前端 (`index.html`)

- **新增 3 个 tab**：Global Execution、Agent Collaboration、Agent Logs
- **Global Execution**：
  - 左侧：运行中卡片 + 待执行任务按 wave 分组
  - 右侧：隔离区面板（quarantine），红色主题
- **Agent Collaboration**：
  - 输入 agent_id → 加载消息 / 标记已读
  - 未读消息高亮（cyan border）
  - 自动轮询：当 tab 激活且输入了 agent_id 时，每 2 秒刷新
- **Agent Logs**：
  - 下拉框从 `agents`、`recentAgents`、`globalStatus.running` 聚合所有 `agent_id`
  - 选中后加载 `activity`（Markdown 渲染）和 `reasoning`（JSONL 卡片列表）
- **响应式布局**：移动端自动堆叠，桌面端 Grid 分栏

### 交互逻辑

- `refreshAll()` 现在并行拉取 `fetchGlobalStatus()`
- `startPolling()` 在 collaboration tab 激活时额外调用 `loadMailbox()`
- `watch(currentTab)` 在切换到新 tab 时按需加载对应数据

---

## Target Impact

Web UI 现在具备完整的全局可观测性，开发者无需手动翻阅 JSONL 或命令行即可查看：
- 哪些 agent 在跨 wave 并发运行
- agent 之间的异步通信
- 每个 agent 的独立日志

这直接提升了 **developer_adoption_rate**，降低了多 Agent 调试门槛。
