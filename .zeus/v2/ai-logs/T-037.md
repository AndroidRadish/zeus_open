# T-037 — Dockerfile + One-Click Start Scripts for v2 Backend Deployment

## Decision Rationale

T-037 is the final task of Wave 12 / US-011 Production Hardening. Its purpose is to make the v2 backend deployable without requiring users to manually remember Python paths, dependency installation steps, or port numbers. A single-command start experience is critical for `developer_adoption_rate`.

Design choices:
- **Dockerfile**: based on `python:3.11-slim` with `git` and `graphviz` pre-installed (needed by workspace bootstrapping and native SVG fallback). A `HEALTHCHECK` is added so orchestrators like Docker Compose or Kubernetes can detect readiness.
- **start.sh / start.ps1**: support both **native Python** mode (fastest for local dev) and **`--build` / `-Build` Docker mode** (for containerized deployment). They auto-detect Python, install missing dependencies, and forward the chosen port.
- **requirements.txt**: the repository previously had no formal dependency manifest. Adding one at the project root makes both native and Docker installs deterministic.
- **No local Docker build verification**: the host Windows environment does not have Docker installed, so the Dockerfile was validated structurally (required directives) and the requirements were validated via `pip install --dry-run`. The scripts were syntax-checked manually.
