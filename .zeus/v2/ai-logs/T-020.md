# T-020 — Implement ZeusOrchestrator._bootstrap_workspace

## Decision Rationale

The subagent dispatcher (M-006) solved *how* to delegate tasks to external AI CLIs, but it left a continuity gap: every subagent started in an isolated workspace with zero context about the project conventions, the human, or its own identity. This forces humans to re-paste `AGENTS.md`, `USER.md`, `IDENTITY.md`, and `SOUL.md` into every subagent session, undermining unattended execution.

The fix is to seed these files automatically before dispatch. The design follows the zero-config principle: if the files exist in `project_root`, copy them; if not, skip silently. An optional `config.json` override (`subagent.bootstrap.files`) lets advanced users customize the list without changing code.

## Execution Summary

- Modified `zeus_orchestrator.py`:
  - Added `_bootstrap_workspace(workspace_path, bus, store)` method.
  - Reads default identity files (`AGENTS.md`, `USER.md`, `IDENTITY.md`, `SOUL.md`) from `project_root` and copies them into the agent workspace.
  - Supports config override via `config.subagent.bootstrap.files`.
  - Emits a `task.bootstrapped` event with `injected` and `skipped` file lists.
  - Integrated the call into `dispatch_task` after source copy and before `PROMPT.md` write.
- Modified `agent_bus.py`:
  - Added `task.bootstrapped` to `SUPPORTED_EVENTS`.
