# T-012 — Refactor orchestrator scheduling loop to priority-queue + adaptive slot filling

## Decision Rationale

The original `await_wave_completion` used a static wave boundary: it loaded all tasks for `wave == N`, created coroutines for all of them, and waited with `asyncio.gather`. This caused concurrency slots to sit idle when one task blocked (e.g., long external dependency) while independent tasks in Wave N+1 were ready to run.

To solve this without breaking backward compatibility, we introduced:
1. **Effective vs. original wave tracking** — `_task_effective_wave()` reads `scheduled_wave` (with fallback to `wave`), while `_task_original_wave()` reads `original_wave` (with fallback to `wave`). This lets the orchestrator distinguish between where a task was planned and where it is actually running.
2. **Priority-queue readiness check** — `_get_ready_tasks()` scans the full task list, filters out already-passed or already-dispatched tasks, checks `depends_on` satisfaction, and applies a `lookahead_waves` window (default 1 for adaptive mode). Results are sorted by `(original_wave, task_id)` so lower-wave tasks still get priority.
3. **Dynamic slot filling** — `await_wave_completion()` now runs a `while True` loop that:
   - Computes free semaphore slots.
   - Pops ready tasks from the heap.
   - If a task’s `original_wave` is greater than the current wave, it is atomically rescheduled via `_reschedule_task()` and a `task.rescheduled` event is emitted to the Agent Bus.
   - Spawns the task via `asyncio.create_task` and waits with `asyncio.wait(return_when=FIRST_COMPLETED)` so new tasks can be injected the moment a slot frees up.
4. **Approval gate on original_wave** — `transition_to_next_wave()` now verifies that all tasks whose `original_wave == current_wave` are passed, regardless of whether they were executed earlier due to rescheduling. This preserves the human approval boundary while allowing runtime optimization.

## Execution Summary

- Added `_task_effective_wave()` and `_task_original_wave()` helpers to `ZeusOrchestrator`.
- Added `_get_ready_tasks()` to compute the ready heap using dependency checks and lookahead policy.
- Added `_reschedule_task()` to atomically update `scheduled_wave`, `wave`, and `rescheduled_from` through `store.update_json_fields()`, plus emit `task.rescheduled` events.
- Added `_run_and_capture()` wrapper so `await_wave_completion` can correlate exceptions with `task_id` after `asyncio.wait`.
- Rewrote `await_wave_completion()` from static `gather()` to dynamic `wait()` loop with on-the-fly slot filling.
- Updated `load_wave()`, `transition_to_next_wave()`, and `print_status()` to use effective/original wave semantics.
- All 5 existing orchestrator tests pass without modification, confirming backward compatibility.

## Target Impact

- `multi_agent_efficiency` ↑↑: concurrency slots are kept full even when individual tasks block.
- `developer_adoption_rate` ↑: reduces human wait time between waves by allowing forward tasks to start early.
- Sets the runtime foundation for the future Auto-heal feature (deferred to backlog), which will reuse the same dynamic loop to dispatch fixer Agents into empty slots.
