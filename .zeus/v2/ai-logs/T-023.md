# T-023 — Add /milestones endpoint and Milestones tab to Web UI

## Decision Rationale

The Dashboard's wave-by-wave dropdown forces users to click repeatedly to understand overall project health. Milestones (M-001, M-002...) are already defined in `roadmap.json` and carry semantic meaning that humans understand naturally. Surfacing milestones as first-class UI citizens eliminates wave-switching friction.

## Execution Summary

- Modified `zeus_server.py`:
  - Added `GET /milestones` endpoint that joins `roadmap.json` with `task.json`.
  - Computes per-milestone `status` (`completed` / `in_progress` / `pending`) and `progress_percent`.
  - Returns milestone tasks with i18n-friendly fields.
- Modified `web/index.html`:
  - Added new tab **里程碑 / Milestones**.
  - Each milestone is a collapsible card with a status badge, progress bar, and expandable task table.
  - Integrated `fetchMilestones` into `refreshAll` and tab watcher.
- Added `test_milestones_endpoint` in `test_server.py` to verify progress calculation and status rules.
- Commit: `b5dd912`

## Target Impact

- `developer_adoption_rate` ↑↑: humans can read project progress in one scroll instead of N wave switches.
- `ui_usability` ↑↑: milestone grouping aligns mental models with the existing roadmap artifact.
