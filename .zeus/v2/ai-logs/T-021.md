# T-021 — Add unit tests for workspace bootstrap behavior

## Decision Rationale

Because `_bootstrap_workspace` runs inside the hot path of every task dispatch, it must be robust: missing files should not raise, existing files must land in the workspace, and config overrides must replace—not augment—the default list. Unit tests are the cheapest way to guarantee this behavior survives future refactors.

## Execution Summary

- Modified `test_orchestrator.py`:
  - `test_dispatch_task_bootstraps_identity_files`: seeds `AGENTS.md` and `USER.md` in the project root, asserts they are copied, asserts missing `IDENTITY.md`/`SOUL.md` are skipped, and verifies the `task.bootstrapped` event payload.
  - `test_dispatch_task_bootstrap_respects_config_override`: sets `config.subagent.bootstrap.files` to `["CUSTOM.md"]`, asserts only `CUSTOM.md` is reported as injected and the default files are ignored by bootstrap logic.
  - Updated `test_dispatch_task_creates_workspace_and_prompt` to expect the new `task.bootstrapped` event in the event stream (3 events instead of 2).
- All 50 v2 tests pass after the changes.
- Commit: `98a0ee9`

## Target Impact

- `developer_adoption_rate` ↑: tested behavior builds confidence for users enabling true unattended execution.
- Prevents regressions in the critical path between workspace creation and subagent dispatch.
