## 🏗️ Architecture Context

### How `run_global` works
```python
orch = ZeusOrchestrator(version="v2", max_parallel=3)
summary = asyncio.run(orch.run_global())
```
- Reads `task.json` each scheduling iteration
- Maintains `dispatched_ids` to avoid double-dispatch
- Uses `quarantine` array to exclude failed tasks from dependency resolution
- CLI flag: `--run-global`

### Where to hook Mailbox in `zeus_orchestrator.py`
The `dispatch_task` method creates a workspace per task. The `agent_id` is currently hardcoded as `f"zeus-agent"`. For T-032, you should make it unique per dispatch, e.g.:
```python
agent_id = f"{dispatcher_name}-agent-{task_id}"
```
