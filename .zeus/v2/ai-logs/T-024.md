# T-024 — Extend task schema and Web UI with multi-language support (EN/ZH)

## Decision Rationale

A previous backlog request for bilingual task metadata was never implemented. With ZeusOpen aiming for universal adoption across Chinese and English-speaking developers, task titles and descriptions must be locale-aware. Adding optional i18n fields to the schema keeps backward compatibility while enabling incremental localization.

## Execution Summary

- Modified `.zeus/schemas/task.schema.json`:
  - Added optional `title_en`, `title_zh`, `description_en`, `description_zh` fields.
- Modified `zeus_server.py`:
  - `/wave/{n}` now returns the i18n fields.
  - `/milestones` also includes them.
- Modified `web/index.html`:
  - Added `lang` ref (`zh` by default).
  - Added a **中 / EN** language toggle pill in the header.
  - Implemented `getLocalized(task, field)` helper that prefers `field_{lang}` and falls back to the base field.
  - Updated Dashboard task list, Milestones task list, and Agents monitor to use `getLocalized` for titles.
- Modified `zeus_orchestrator.py`:
  - `_build_prompt` now uses `_localize(task, "title", "zh")` and `_localize(task, "description", "zh")` so Chinese prompts benefit from `title_zh`/`description_zh` when available.
- Commit: `b5dd912`

## Target Impact

- `developer_adoption_rate` ↑↑: bilingual metadata lowers the barrier for mixed-language teams.
- `ui_usability` ↑: instant language toggle without page reload.
