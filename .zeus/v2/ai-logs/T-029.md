# T-029 — Add tests for /phases and phase status rules; update documentation

## Decision Rationale

Any new API endpoint needs automated verification to prevent regressions. The `/phases` endpoint involves multi-file joins (`roadmap.json` + `task.json`) and derived state (status, progress), making it especially important to test. Documentation ensures users know how to define phases without reading source code.
