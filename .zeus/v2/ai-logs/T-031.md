# T-031 — Implement Agent Mailbox protocol in agent_bus.py

**Commit SHA:** `1b02a1d6d6e59104d7416e891b103d0f92deb1a6`

---

## Decision Rationale

为了保持 ZeusOpen v2 的 Agent 协作能力可扩展，我们需要一个轻量级的 Mailbox 协议：
- **持久化**：消息以 JSONL 形式存储在 `.zeus/v2/agent-logs/mailbox/{to_agent_id}.jsonl`，便于文件系统级别的观察和 tail。
- **API 设计**：`send(to_agent_id, message)` 和 `receive(agent_id, mark_read=False)` 足够表达一对多的异步通信。
- **集成**：`zeus_server.py` 新增 `GET /mailbox/{agent_id}` 端点，支持 Web UI 的 Agent Collaboration 视图（T-033 的前置条件）。

---

## Execution Summary

### 修改的文件

1. **`.zeus/v2/scripts/agent_bus.py`**
   - `AgentBus.__init__` 新增可选 `agent_id` 参数。
   - 新增 `send(to_agent_id, message)`：将消息追加到对应 agent 的 mailbox JSONL。
   - 新增 `receive(agent_id, mark_read=False)`：读取并可选地标记已读状态。

2. **`.zeus/v2/scripts/zeus_server.py`**
   - 新增 `GET /mailbox/{agent_id}` 端点，支持 `mark_read` 查询参数。

3. **`.zeus/v2/scripts/test_agent_bus.py`**
   - 新增 `test_send_and_receive_mailbox` 和 `test_receive_empty_mailbox`。

4. **`.zeus/v2/scripts/test_server.py`**
   - 新增 `test_mailbox_endpoint` 和 `test_mailbox_endpoint_empty`。

---

## Target Impact

Mailbox 协议为后续 T-033（Agent Collaboration UI）和全局调度器提供了事件外的基础设施，直接提升 **developer_adoption_rate**：Agent 间的可观测通信降低了多 Agent 项目的调试成本。
